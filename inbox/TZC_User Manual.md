##  1.1 TZC简介

本节介绍了TrustZone控制器，该控制器包含以下子模块：

- 主机状态控制器

子模块定义了主机的安全，特权状态。

- 外设状态控制器

子模块定义了外设的安全，特权状态。

- 非法访问控制器

子模块收集系统中的所有安全违例访问事件，并针对NVIC生成安全中断。

## 1.2 TZC的主要特性

TZC的主要特性如下：

- 一个32为APB接口，用于TZC寄存器配置
- 非法访问控制器仅限安全特权访问
- 主机状态控制器和外设状态控制器的读访问不受限
- 主机状态控制器和外设状态控制器的安全状态配置仅限安全特权访问。
- 主机状态控制器和外设状态控制器的特权状态配置受受对应的安全状态配置控制。
	- 安全：仅限安全特权访问。
	- 非安全：仅限非安全特权访问。

**GTZC TrustZone 系统架构**

Armv8-M 根据 TrustZone-M 型号提供安全支持，并采取以下隔离措施：

- 安全环境，通常运行安全敏感型应用程序，并存放关键资源
- 非安全或公共环境（例如：非安全操作系统和用户空间）

TrustZone 架构在 AHB 和 Armv8-M 基础上进行了扩展，具有：

- MSC（主机安全控制器）是一种适配器，用于将不具备安全感知能力的总线主机接入支持 TrustZone 的系统，为该主机分配安全属性，并依据 IDAU 定义的地址属性对每一笔事务进行安全检查与强制约束。
- PPC（外设保护控制器），用作安全门，可阻止或向 AHB 代理传播安全/非安全和特权/
  非特权事务
- MPC（存储保护控制器），用作安全门，用于过滤针对内部 SRAM 的安全/非
  安全、特权/非特权访问

AXI/AHB/APB 外设可分为：

- TrustZone非感知：通过安全控制器来定义外设安全和特权属性
- TrustZone感知：直接连接到AXI/AHB/APB总线、模块内部实现安全和特权属性
- 安全：外设始终为安全属性。
- 非安全和非特权：直接连接到 AXI/AHB/APB互连的外设，没有任何安全门

可在TZC中将TrustZone非感知外设配置为安全，非安全和特权，非特权。

## 1.4 TZC功能说明

### 1.4.1 TZC 框图

下图说明了MAC，PAC，VIC的组合功能，三个子模块共用APB配置端口控制。

主机的安全或特权配置位必须通过安全特权事务修改。

外设的安全配置位必须通过安全特权事务修改。

当外设为安全属性时，特权配置位必须通过安全特权事务修改。否则，特权配置位可以由安全或非安全特权事务修改。

检测到非法访问时，TZIC会收集系统内生成的非法事件。然后，TZIC可根据需要针对CPU生成一个安全中断。

![TZC_Block_Diagram](https://pic.lllincx.cn/TZC_Block_Diagram.png)

### 1.4.2 非法访问定义

存在三种非法访问类型：

**非法的非安全访问**

任何试图读写安全资源的非安全事务都被视为非法，因此被寻址的资源针对非法读写访问生成非法访问中断。

**非法安全访问**

任何试图读写非安全资源的安全事务都被视为非法，因此被寻址的资源针对非法读写访问生成非法访问中断。

**非法的非特权访问**

任何试图访问特权资源的非特权事务都被视为非法。针对非法读写访问不生成非法访问事件。被寻址的资源遵循静默失败行为，对于读操作，返回全零数据，并忽略任何写操作。不会生成总线错误。任何非特权执行事务试图访问特权存储器时，都会生成非法访问中断。 

### 1.4.3 主机属性控制器（MAC）

MAC由13组可配置的寄存器组成，TZC_MASTER_CFG_R<x>寄存器用于控制表格所示主机的安全、特权属性。

- <master_name>_priv位表示主机的特权属性。0：非特权，1：特权。
- <master_name>_sec位表示主机的安全属性。0：非安全，1：安全。

TZC_MASTER_CFG_R<x>寄存器读取不受限，写入仅限安全特权事务。

dubhe主机的安全属性不由tzc配置。

> 见表1.1

TZC控制信号与主机MSC并非一一对应关系，如表1.2所示，TZC控制信号同时控制多个主机。

### 1.4.4外设属性控制器（PAC）

PAC由两组寄存器组成，每组包括三个寄存器，其中一组寄存器控制外设的安全属性，另一组控制特权属性。寄存器每个位域一一对应。

TZC_SEC_CFG_R<x>寄存器用于控制表格所示外设的安全属性。

<peri_name>_sec位表示对应外设的安全属性。0：非安全，1：安全。

TZC_SEC_CFG_R<x>寄存器读取不受限，写入仅限安全特权事务。



TZC_PRIV_CFG_R<x>寄存器用于控制表格所示外设的特权属性。

- <peri_name>_priv位表示外设的特权属性。0：非特权，1：特权。

TZC_PRIV_CFG_R<x>寄存器读取不受限。写入权限受TZC_SEC_CFG_R<x>寄存器影响。

- TZC_SEC_CFG_R<x>[<peri_name>_sec]=0：写入限特权事务，不限安全属性。
- TZC_SEC_CFG_R<x>[<peri_name>_sec]=1：写入仅限安全特权事务。

> 见表2.1

TZC控制信号与外设并非一一对应关系，如表2.2所示，TZC控制信号同时控制多个外设。

### 1.4.5 违例中断控制器（VIC）

VIC 汇总所有非法访问源事件。

VIC 允许跟踪（标志）触发安全非法访问中断的事件。屏蔽寄存器 (TZC_INTR_EN_R<x>) 可
用于过滤不需要的事件。对于未屏蔽的非法事件，VIC 生成 NVIC 的 TZC_IRQ 中断。
对于每个非法事件源，存在一个状态标志和一个清零位（分别在TZC_INTR_SS_R<x> 和
TZC_INTR_CLR_R<x> 内）。屏蔽寄存器 (TZC_INTR_EN_R<x>) 的复位值用于屏蔽所有事件。

包括4.3和4.4章节的主机和外设之外，表中的MPC和安全感知外设也会输出安全违例中断到VIC。

> 见表3

TZC接收的中断请求与外设并非一一对应关系，如表2.2所示，多个外设的中断请求或处理后输入TZC。

### 1.4.6 上电，复位状态

TZC 的上电和复位状态将 TZC_MASTER_CFG_R<x>，TZC_SEC_CFG_R<x>和TZC_PRIV_CFG_R<x>的所有位都清零，这意味着所有受控主机和外设分别设置为非安全和非特权。

## 1.5 TZC中断

TZIC 是安全外设，因此在发生非安全访问时，在系统级生成非法访问事件。MPCBB 和
TZSC 是 TrustZone 感知外设，这意味着安全和非安全寄存器同时存在于外设中。

| 中断缩写 | 中断事件 | 事件标志                     | 使能控制位                 | 中断清除方法                |      |
| -------- | -------- | ---------------------------- | -------------------------- | --------------------------- | ---- |
| TZC      | 非法访问 | TZC_INTR_SS_R<x>中的所有标志 | TZC_INTR_EN_R<x>中的所有位 | 向TZC_INTR_CLR_R<x> 位写入1 |      |

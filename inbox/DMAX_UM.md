# 1. DMAX简介

AXI总线接口直接存储器访问控制器是(DMAX)一个主控总线系统外设。
当通过控制为 CPU 减负时，可以使用 DMAX 通过链表在存储器映射的外设和/或存储器之间执行可编程的数据传输。



# 2. DMAX主要特性

```
可编程 APB4 配置寄存器

AXI5 数据路径

用于时钟和电源的低功耗接口（LPI）

1-8 个并行 DMA 通道，每个通道带有可配置的 FIFO

包含地址增量的 1D 存储器复制

每个通道的中断能力及全局事件中断

灵活的命令链接（Command Linking）能力

用于流控制的触发能力

扩展存储器复制能力：2D/回环（Wrap）/模板（Template）

扩展接口
```



- 两组双向AXI主接口
- 从源到目标的内存映射数据传输：
	- 外设到内存
	- 内存到外设
	- 内存到内存
	- 外设到外设
- 基于通道级别 16级可编程优先级的传输仲裁： 采用 LRG (Least-Recently Granted) 算法，结合固定优先级与轮询（Round-robin）两层架构
- 每通道事件生成，可针对以下任一事件触发：等待触发输出，等待目标触发输入，等待源触发输入，等待恢复，暂停，停止，禁用，错误，完成。
- 每通道中断生成，每个事件可单独编程使能中断
- 8 个并发 DMAX 通道：
	- 每个通道独立的FIFO
	- 通过可编程链表（linked-list）进行通道内 HPDMA 传输链接，将链表存入内存，支持两种执行模式：运行至完成（run-to-completion）和链接步进模式（link step mode）
	- 通过将可编程 HPDMA 输入触发连接到 HPDMA 任务完成事件，实现通道内和通道间 HPDMA 传输链接
- 通道内每个链表项（linked-list item）：
	- 单独编程的源和目标传输
	- 源与目标之间可编程的数据处理：基于字节的重排序、打包或解包、填充或截断、符号扩展以及左/右重对齐
	- 可编程的源端待传输数据字节数，定义块（block）级别
	- 线性源和目标寻址：在连续突发传输之间，于块级别编程为固定或连续递增寻址

------

*High-performance direct memory access controller (HPDMA)* *RM0486* *828/4691* *RM0486 Rev 2*

------

- 二维（2D）源和目标寻址：在连续突发传输之间可编程的有符号地址偏移（块内非连续寻址），并在第二个 2D/重复块级别上结合连续块之间可编程的有符号地址偏移，适用于部分通道（参见 18.3.1 节）
- 通过 2D 寻址支持分散-聚集（scatter-gather，多缓冲区传输）、数据交织（interleaving）和解交织（deinterleaving）
- 可编程 HPDMA 请求和触发的选择
- 可编程 HPDMA 半传输和传输完成事件的生成
- 指向内存中下一个链表项及其数据结构的指针，并自动更新 HPDMA 链表控制寄存器
- 通道中止和重启
- 调试：
	- 通道挂起和恢复支持
	- 通道状态报告，包括 FIFO 级别和事件标志
- TrustZone 支持：
	- 支持安全和非安全 HPDMA 传输，在第一个通道级别上独立，并在源/目标和链接子级别上独立
	- 安全和非安全中断报告，分别由相应的安全和非安全通道产生
	- 具备 TrustZone 感知能力的 AHB 从端口，保护任何 HPDMA 安全资源（寄存器、位域）免受非安全访问
- 特权/非特权支持：
	- 支持特权和非特权 HPDMA 传输，在通道级别上独立
	- 具备特权感知能力的 AHB 从端口
- 通道隔离支持：
	- 支持分隔（compartmented）DMA 传输，在通道级别上独立，通过分隔 ID（称为 CID）实现
	- 具备 CID 感知能力的中断报告
	- 具备 CID 感知能力的 AHB 从端口，内置信号量（semaphore）以支持来自任一 CPU 的并发控制

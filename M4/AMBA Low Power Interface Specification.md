## 1.1 关于低功耗接口

本文档描述了可用于控制设备时钟和电源状态的接口。这些接口包括：

- **Q-Channel**：适用于简单的运行-停止（run-stop）静默语义场景。Q-Channel 是 AXI 低功耗接口的演进，其命名反映了该接口的目的是控制设备的静默（quiescence）。

- **P-Channel**：适用于具有多种转换的复杂电源管理场景。这些转换无需将设备恢复到通用可操作状态即可完成。其命名反映了该接口的目的是控制电源状态转换。

> **注意**：每个接口均具有可选的接口奇偶校验保护（Interface Parity Protection）。该保护提供了检测、诊断和安全缓解接口上潜在故障的能力。

### 1.1.1 共同特性

Q-Channel 和 P-Channel 接口具有以下共同特性：

- 由控制器管理的设备状态间转换。
- 设备可以：
  - 指示其必须退出较低功耗状态并进入较高功能状态。
  - 提示其可能接受进入较低功耗状态的请求。
- 可选地，设备可以拒绝状态更改请求。
- 跨时钟域语义具有鲁棒性，以支持异步接口。

尽管这些接口旨在用于时钟和电源状态控制，但对接口所管理的状态的具体特性没有限制。
## 1.2 接口差异与选择

这两种接口被设计为互补关系。接口的选择取决于具体实现的需求。

### 1.2.1 Q-Channel

Q-Channel 简单的运行-停止静默语义非常适合时钟控制或简单的电源控制场景。进入的静默状态可以不同，但仅限于设备和控制器在进入静默状态前的共同配置。静默状态的退出转换始终回到一个通用的可操作运行状态，且必须先进入该状态后，设备才能进入另一个不同的静默状态。

### 1.2.2 P-Channel

P-Channel 可以处理更复杂的场景。可以在不将设备或设备分区恢复到通用可操作状态的情况下，在连续的省电状态之间进行多次转换。控制器在每次转换时提供明确的状态请求，因此不需要控制器和设备之间的共同配置。

仅当 Q-Channel 功能不足时，才应使用 P-Channel 接口。
## 2.1 Q-Channel 接口规范

Q-Channel 是 AXI 低功耗接口的演进，在大多数情况下向后兼容。更多信息请参阅第 2-37 页"Q-Channel 向后兼容性"。

Q-Channel 通过使握手机制独立于设备活动指示，简化了跨时钟域处理。

图 2-1 展示了设备与时钟或电源控制器之间的信号：

**图 2-1 Q-Channel 设备与控制器信号**

Q-Channel 接口具有以下独立的信号组。

#### 设备活动指示

该组包含单一信号 `QACTIVE`，设备可在任何状态下将其驱动为高电平，以指示其有操作需要执行。当设备将 `QACTIVE` 驱动为低电平时，这仅是一个提示而非保证，表明设备可能会接受静默请求。

设备的 `QACTIVE` 信号可由多个源信号组合而成。为提供唤醒能力，这些源信号可包含设备输入信号。最终的 `QACTIVE` 信号要么直接由寄存器驱动，要么由一个或多个寄存器和输入信号经逻辑组合后驱动。第 2-31 页"Q-Channel 实现"给出了 `QACTIVE` 组合逻辑的实现建议。

如果设备本身在缺少由该接口管理的时钟或电源供应时无法将 `QACTIVE` 置为高电平，则必须有一种系统级方法在设备外部实现唤醒。例如，可以将另一设备的 `QACTIVE` 在控制器端与该设备的 `QACTIVE` 进行组合。

#### 握手机制

该组管理设备静默并保证安全的状态转换。握手接口包含：

- 静默请求信号 `QREQn`，由控制器驱动。
- 应答信号对 `QACCEPTn` 和 `QDENY`，由设备驱动回控制器以指示接受或拒绝请求。应答信号的组织方式确保每次握手转换仅有其中一个信号发生变化，从而保证接口可以在异步边界上安全实现。`QACCEPTn` 用于接受请求，`QDENY` 用于拒绝请求。

设备的 `QACCEPTn` 和 `QDENY` 信号以及控制器的 `QREQn` 信号都必须由寄存器驱动。

拒绝机制的目的是使设备能够在保持运行状态的同时，拥有一种可以快速完成握手的机制。

握手信号的极性被组织为在静默状态下所有接口信号均为低电平，这有利于简化默认隔离规则。

## 3.1 P-Channel 接口规范

本节描述控制电源状态转换的 P-Channel 接口。在本规范中：

- **较低功耗状态（Lower-power state）**：通常指功耗较低、设备功能性、性能或状态保持能力低于较高状态的状态。
- **较高功耗状态（Higher-power state）**：通常指功耗较高、设备功能性、性能或状态保持能力高于较低状态的状态。

### 3.1.1 P-Channel 信号

图 3-1 展示了设备与电源控制器之间的 P-Channel 信号。

**图 3-1 P-Channel 设备与控制器信号映射**

P-Channel 接口包含以下信号组。

#### 设备活动指示

该组包含一个 N 位信号 `PACTIVE[N-1:0]`：

- 呈现在控制器端的每个 `PACTIVE` 位可由多个源信号组合而成。为提供唤醒能力，这些源信号可包含设备输入信号。每个 `PACTIVE` 位要么直接由寄存器驱动，要么由一个或多个寄存器和输入信号经逻辑组合后驱动。第 3-57 页"P-Channel 实现"给出了 `PACTIVE` 组合逻辑的实现建议。
- `PACTIVE` 的所有位彼此独立，可能来自不同的异步源。更多信息请参阅第 3-57 页"P-Channel 实现"。

#### 握手机制

该组管理设备电源状态转换，包含以下信号：

- **`PSTATE[M-1:0]`**：请求转换到的目标电源状态。
- **`PREQ`**：高电平有效，请求转换到 `PSTATE` 所指示的电源状态。
- **`PACCEPT`**：高电平有效，接受向所请求电源状态的转换。
- **`PDENY`**：高电平有效，拒绝电源状态转换请求。

`PREQ`、`PACCEPT` 和 `PDENY` 构成握手接口，用于管理并保证安全的状态转换。该握手接口包含由控制器驱动的转换请求信号 `PREQ`，以及由设备驱动的应答信号对 `PACCEPT` 和 `PDENY`，用于指示接受或拒绝请求。

P-Channel 规范要求在单次握手转换中仅有 `PACCEPT` 或 `PDENY` 之一发生变化，从而保证接口可以在异步边界上安全实现。`PACCEPT` 用于接受请求，`PDENY` 用于拒绝请求。

设备的 `PACCEPT` 和 `PDENY` 信号以及控制器的 `PREQ` 和 `PSTATE` 信号都必须直接由寄存器驱动。`PSTATE` 的枚举值由具体实现定义（IMPLEMENTATION DEFINED），建议使用 Power Policy Unit Specification 中定义的值以保持兼容性。更多信息请参阅 *Arm® Power Policy Unit, Architecture Specification, Version 1.1*。

拒绝机制的目的是使设备能够维持当前状态，同时拥有一种可以快速完成握手的机制。这意味着如果系统条件发生变化，控制器可以发出另一个请求，可能指向不同的状态。

握手信号的极性被组织为在每次转换的起始和结束时均为低电平，这有利于简化默认隔离规则。

握手信号的状态独立于 `PACTIVE` 位。`PACTIVE` 位上的转换不受 `PREQ`、`PACCEPT` 和 `PDENY` 值的约束。

P-Channel 接口还与设备或设备分区的复位存在协议关系。对于具有多个复位的设备，此处指定的复位关系是与设备 P-Channel 逻辑相关联的复位。本规范中使用 `RESETn` 表示该复位信号。但实际的复位命名和极性不在本规范范围内，只要遵守所描述的关系即可。第 3-43 页"设备复位与初始化"描述了复位和初始化转换。

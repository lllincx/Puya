# Abstract

访问安全地址：

1. 存在寄存器 `SPCSECCTRL` 锁定所有安全、特权配置寄存器
2. PPC安全、非安全访问配置寄存器：1. 仅非安全访问。2.仅安全访问。（待确定）
3. PPC特权、非特权访问配置寄存器：1. 安全特权，安全非特权均可访问。2. 仅安全特权访问。

访问非安全地址：

1. PPC特权、非特权访问配置寄存器：1. 非安全特权非安全非特权均可访问。2. 仅非安全特权访问。

# SPEC

## 3.1 系统与安全控制

SSE-320 在系统中提供了多个寄存器，用于发现、配置和控制系统的各种功能。

这些寄存器被分为四个寄存器块：

**系统信息寄存器块** 系统信息寄存器块提供系统配置与标识信息。

**系统控制寄存器块** 系统控制寄存器块实现了电源、时钟、复位及其他通用系统控制相关的寄存器。

**安全访问配置寄存器块** 安全访问配置寄存器块提供用于配置外设保护控制器（PPC）和管理器安全控制器（MSC）的寄存器，这些控制器位于系统内部以及通过安全控制扩展接口连接的扩展系统中。这些寄存器仅允许安全（Secure）访问。

**非安全访问配置寄存器块** 非安全访问配置寄存器块提供用于配置外设保护控制器（PPC）的寄存器，这些控制器位于系统内部以及通过安全控制扩展接口连接的扩展系统中。这些寄存器仅允许非安全（Non-secure）访问。

### 3.1.1 外设保护控制器

系统中的外设保护控制器（PPC）使软件能够控制外设是否可被安全世界或非安全世界访问，并控制特权访问或非特权访问。

更多信息请参阅 _Arm® CoreLink™ SIE-200 嵌入式系统 IP 技术参考手册_和 _Arm® CoreLink™ SIE-300 AXI5 嵌入式系统 IP 技术参考手册_。在 SSE-320 中，被映射到两个内存区域（一个安全区域、一个非安全区域）的外设由 PPC 保护。PPC 定义外设所处的区域。

在 SSE-320 中，定时器、NPU 及其他外设均通过 PPC 进行保护，由安全访问配置寄存器块和非安全访问配置寄存器块控制。

这些寄存器还控制安全控制扩展信号，以驱动外部 PPC。

SSE-320 中定义了两组受外设保护控制器保护的外设：

- **外设互连外设保护控制器组 0（PIPPC0）**，包含以下外设：

	- 所有基于时间戳的定时器

	- NPU0

	- 看门狗刷新帧

	- 内存保护控制器配置寄存器块

	- 安全访问配置寄存器块

	- 非安全访问配置寄存器块

	- 看门狗控制帧

	- 调试系统访问

	- SDC-600 内部

	> 看门狗刷新帧的安全或非安全映射是固定的，仅其特权级别可配置。

- **外设互连外设保护控制器组 1（PIPPC1）**，包含以下外设：

	- SLOWCLOCK CMSDK 定时器

	- 系统控制寄存器块

	- SLOWCLOCK 看门狗定时器

	- PPU

	- 生命周期管理器（当 LCM_KMU_SAM_PRESENT = 1 时）

	- 密钥管理单元（当 LCM_KMU_SAM_PRESENT = 1 时）

	- 安全告警管理器（当 LCM_KMU_SAM_PRESENT = 1 时）

	> 如果允许非特权访问系统中的总线管理器（例如 DMA），应格外谨慎。若允许非特权访问，则非特权代码有可能利用这些管理器访问和修改特权内存。Arm 建议仅允许特权编程访问这些管理器。


	**相关信息**

- 外设互连外设保护控制器扩展，第 157 页

- 电源域，第 96 页

### 3.1.2 管理器安全控制器

系统中的管理器安全控制器（MSC）将面向 A 类系统设计的非 CPU 管理器所发出的内存事务，转换为适用于 M 类系统的内存事务。

更多信息请参阅 _Arm® CoreLink™ SIE-200 嵌入式系统 IP 技术参考手册_和 _Arm® CoreLink™ SIE-300 AXI5 嵌入式系统 IP 技术参考手册_。

**相关信息**

- 管理器安全控制器扩展，第 160 页

### 3.1.3 内存保护控制器

系统中的内存保护控制器（MPC）将内存模块划分为页，并允许软件定义每个区域是安全的还是非安全的。

在 SSE-320 中，每个受 MPC 保护的内存页被映射到两个内存区域：

- 安全区域
- 非安全区域

根据软件在 MPC 中为该页定义的安全属性，该页仅存在于安全区域或非安全区域中。

每个 VM 配备一个独立的 MPC，每个 MPC 位于外设区域所定义的主内存中。

更多详情请参阅 _Arm® CoreLink™ SIE-200 嵌入式系统 IP 技术参考手册_和 _Arm® CoreLink™ SIE-300 AXI5 嵌入式系统 IP 技术参考手册_。

## 4.12 安全控制扩展信号

SSE-320 提供安全控制信号，用于支持扩展系统中的额外控制器。

这些控制器包括：管理器安全控制器（MSC）、存储器保护控制器（MPC）、外设保护控制器（PPC），以及带有写缓冲的桥接器。

这些信号允许你通过子系统中的安全控制寄存器来控制所有组件。你可以使用配置渲染参数来禁用其中的许多接口，甚至可以禁用单个位。禁用接口的某个位或整个接口并不会移除该接口本身，但会移除相关逻辑，并将 `NSMSCEXP` 输出中未使用的位拉高（HIGH），将其他安全扩展接口输出中未使用的位拉低（LOW）。

更多信息请参见"可配置渲染选项"中用于配置安全接口的可配置渲染参数。

> - 任何未使用的输入信号必须拉低（LOW）。
> - 不支持脉冲类型中断。脉冲中断在连接到该接口之前，必须转换为电平类型中断。

所有安全控制扩展信号与 `SYSSYSCLK` 同步，相应的 Q-Channel 接口控制 `SYSSYSCLK` 的可用性。

> 当任何安全扩展中断的 CLEAR 处于挂起状态时，SSE-320 会置位内部 `SYSCLKQACTIVE` 并拒绝进入静默状态的请求，从而使 `SYSSYSCLK` 保持运行。

这些信号位于 `PD_SYS` 电源域中。

==除 MPC 外，所有中断 STATUS 输入均为电平中断，必须保持置位状态，直到 SSE-320 将相应的中断 CLEAR 输出拉高（HIGH）。对应的 STATUS 和 CLEAR 信号必须实现**四相握手协议**。

==对于 MPC，中断 STATUS 输入为电平中断，必须保持置位状态，直到 SSE-320 通过其编程接口清除 MPC 中相应的中断。

子系统将每个中断源总线进行合并和寄存，并将寄存后的中断路由至**外部唤醒中断控制器（EWIC）**。EWIC 位于 Cortex-M85 或 Cortex-M55 处理器外部，驻留在 `PD_AON` 电源域中，负责将处理器从 OFF 状态唤醒。从 EWIC 出发，IRQ 被路由至相应 Cortex-M85 或 Cortex-M55 处理器内的**嵌套向量中断控制器（NVIC）**。

对任何系统的调试访问遵循正常的存储器映射和译码规则，但如果调试器执行的操作在正常情况下会引发安全违规中断，则不得因此产生安全违规中断。SSE-320 在下级扩展接口上提供 DAI（调试访问标识）信息，可用于识别调试访问，从而在子系统外部集成的扩展逻辑中禁用 MPC 和 PPC 的中断产生。

# PKA

### 地址错误

对 PKA RAM 的访问超出预期范围。
置位 `PKA_SR` 寄存器中的`ADDRERRF`位域

### RAM 访问错误

当 PKA 内核正在使用 PKA RAM 时，发生了 AHB 总线对该 RAM 的访问。
置位 `PKA_SR` 寄存器中`RAMERRF`位域。读零写忽略。

### 操作错误

`MODE` 位域选择的操作模式不在 PKA 支持列表中（详见位域描述），或 PKA 运行在受限模式下（见 `PKA_SR` 中的 `LMF` 位）。
置位 `PKA_SR` 寄存器中`OPERRF`位域。读零写忽略。

## CRYP

### 密钥错误

置位 `KERF` 标志；清除 `CRYP_SR` 寄存器的 `KEYVALID` 位。

- 写入顺序错误：检测到错误的密钥寄存器写入序列。
- 共享大小不匹配错误：当 `KMOD[1:0]` 为 `0x2` 且 CRYP 中的 `KEYSIZE` 与 SAES 外设中的 `KEYSIZE` 不匹配，或 CRYP 中的 `KEYSIZE` 为 `0x1`（192 位密钥）。
- 共享错误：SAES 共享密钥传输至 CRYP 失败时触发。

## SAES

### 读错误

对 `SAES_DOUTR` 寄存器进行非预期的读操作，计算阶段或输入阶段触发。
置位 `RDERRF` 标志和 `RWEIF` 标志，读零。
若 `RWEIE` 位已设置，则会产生中断。

### 写错误

对 `SAES_DINR` 寄存器进行非预期的写操作，计算阶段或输出阶段触发。
置位 `WRERRF` 标志和 `RWEIF` 标志，写忽略。
若 `RWEIE` 位已设置，则会产生中断。

### 密钥错误

置位 `SAES_ISR` 寄存器的 **KEIF** 标志，并清除 `SAES_SR` 寄存器的 `KEYVALID` 位

- 密钥保护错误：当 `KEYVALID` 已置位，且 `KEYPROT` 已置位或 `KEYSEL[2:0]` 不为零时，程序尝试访问 SAES 时安全，则触发此错误。
- 硬件机密密钥加载错误：DHUK 或 BHK 加载至 SAES 失败时触发。此时 `KEYSEL[2:0]` 为 `0x1` (DHUK)、`0x2` (BHK) 或 `0x4` (DHUK XOR BHK) 的配置将无法工作。
- 写入顺序错误：检测到错误的密钥寄存器写入序列。
- 共享大小不匹配错误：当 `KMOD[1:0]` 为 `0x2` 且 CRYP 中的 `KEYSIZE` 与 SAES 外设中的 `KEYSIZE` 不匹配，或 CRYP 中的 `KEYSIZE` 为 `0x1`（192 位密钥）。
- 共享错误：SAES 共享密钥传输至 CRYP 失败时触发。

### RNG 错误

从 RNG 外设获取随机数时（例如因熵值不足）检测到错误
置位 `SAES_ISR` 寄存器的 `RNGEIF` 标志。

### 入侵错误

发生非预期硬件故障时，触发`saes_itamp_out`接口，发送信号给 TAMP。触发 TAMP 外设中的 SAES 内部入侵事件，并停止任何 SAES 协处理器的处理工作，SAES 自动清除密钥寄存器。
要恢复正常运行，需通过 RCC 或全局复位来复位 SAES 外设。

## RNG

### 时钟错误

当时钟错误检测使能（`CED = 0`），且 RNG 时钟频率过低
`CEIS`，`CECS`置 1，指示发生了时钟错误。

### 噪声源错误

噪声源（或种子）错误
停止生成随机数， `SEIS` 和 `SECS` 置 1。

如果此时 `RNG_DR` 寄存器中已有数值，严禁使用该数值，因为其熵值可能不足。

- 当 `ARDIS` 清零（开启自动复位）时： 在重新获取随机数之前，直接清除 `RNG_SR` 寄存器中的 SEIS 中断状态位即可。
- 当 `ARDIS` 置位（禁用自动复位）时，需按此顺序执行：
  1. 在 `RNG_CR` 寄存器中将 `CONDRST` 先置 1 再置 0。
  2. 等待硬件自动清零` CONDRST` 位，然后清除 `RNG_SR` 中的 `SEIS` 中断状态位。
  3. 如果 `RNG_SR` 中的 `SECS` 仍然置位，说明发生了新的种子错误（极少见），此时返回步骤 1 重新操作。

### 入侵错误

当 RNG 发现非预期错误时，会触发 TAMP 外设中的内部入侵（Tamper）事件，且 RNG 停止输出随机数据。

一旦发生此事件，安全应用程序需要通过中央复位管理或 SoC 全局复位来**复位 RNG 外设**。之后，需要重新对 RNG 进行正确的初始化。
